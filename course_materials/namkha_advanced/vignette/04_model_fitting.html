2 Introduction

This tutorial fits four related advanced model sets. The full-survey and shortened-survey analyses are closest to the basic model-fitting workflow, but they use the elevation-constrained fitting mask and gorge-avoiding movement distances. The multisession analysis uses a list of capture histories, masks, and userdist objects. The sex-specific analysis demonstrates two ways to include individual sex information.

The model sets are deliberately small and explicit. This keeps the teaching focus on how the advanced inputs change the fitting call, rather than on a large model-selection exercise.

3 Preparations

Only secr is needed for fitting because the capture histories, masks, and user-distance objects have already been created.

# Packages needed

4 Encounter and detection functions

The advanced models use the hazard half-normal detection function, detectfn = "HHN". With count detectors, it is often more useful to think about the encounter-rate function, \(\lambda(d)\), because an individual can be detected more than once during an occasion. Detection probability, \(g(d)\), is then derived from the encounter rate.

For the hazard half-normal model, encounter rate declines with distance from an activity centre to a camera. lambda0 is the encounter-rate parameter at distance zero, and sigma controls the spatial scale over which encounter rate declines. Both are fitted on a link-function scale, so we usually interpret predictions on the natural scale rather than reading raw coefficients directly.

Trap covariates such as cliff and valley_stream enter the lambda0 model. Mask covariates such as std_tri and std_elev enter the density model, D. Keeping those roles distinct is especially important in the advanced tutorial because the mask and detector covariates come from different spatial objects.

5 Full-survey models

The full-survey model set is the closest advanced equivalent to the basic model-fitting tutorial. The two important changes are that the fitting mask is mask_model, which has been clipped to plausible elevations, and that details = list(userdist = userd_model) tells secr.fit() to use gorge-avoiding movement distances.

When fitting many models, it is good practice to check whether the optimiser finished cleanly and whether any coefficient estimates are NA or infinite. trace = FALSE suppresses the iteration output from secr.fit(), but it does not remove the need to inspect warnings and fitted objects.

The fitted coefficients returned by coef() are on the link scale. For positive parameters such as lambda0 and sigma, the natural-scale estimates are obtained by applying the inverse link function. predict() does this transformation for the fitted model and is usually the safer way to inspect interpretable parameter estimates.

8 Sex-specific full-survey models
