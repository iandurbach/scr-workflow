2 Introduction

In this tutorial we fit a small set of spatial capture-recapture models using secr.fit(). The aim is to show the mechanics of fitting models with the objects created in the previous tutorials: the capture history ch, the trap covariates attached to traps, and the spatial covariates attached to mask_model.

An SECR model has a density component and a detection component. The density component describes how expected animal density varies across the mask. The detection component describes how the encounter rate changes with distance from each detector and, optionally, with detector-level covariates.

The models here are intentionally written out explicitly. This makes it clear which parameter is being modelled in each candidate model and avoids hiding the main SECR ideas inside loops or helper functions.

3 Preparations

We only need the secr package for this script because the data preparation has already been done.

# Packages needed

output_file <- "tutorials/namkha_basic/output/namkha_basic_fitted_models.RData"

In the models below:

  • D is density, modelled using mask covariates such as std_tri and std_d2hydro
  • lambda0 is the baseline encounter-rate parameter, modelled using trap covariates such as valley_stream and cliff
  • sigma controls the spatial scale of detection

5 Encounter and detection functions

The models use the hazard half-normal detection function, specified with detectfn = "HHN". For count detectors, it is helpful to think in terms of an encounter-rate function: the expected number of encounters declines as the distance between an animal’s activity centre and a camera increases.

lambda0 is the encounter-rate parameter at distance zero, and sigma controls how quickly encounter rate declines with distance. Both parameters must be positive, so secr fits them on a link-function scale. We usually interpret fitted values and predictions on the natural scale rather than directly interpreting the raw coefficients.

6 Fit candidate models

The fitting mask is mask_model, not the Namkha boundary mask. This allows animals with activity centres outside Namkha to contribute to the likelihood if they are still within range of traps near the boundary. Abundance and density can later be summarised over the survey-area mask.

6.1 Null model

The null model has no density covariates and no detector covariates. It is the baseline model against which the other candidate models are compared.

# The fitting mask is the trap-buffer mask, not the Namkha boundary mask.
# This allows activity centres outside Namkha if they are within range of traps.

# Null model: no density or detector covariates
m0 <- secr.fit(
  ch,
  detectfn = "HHN",
  mask = mask_model,
  model = list(D ~ 1, lambda0 ~ 1, sigma ~ 1),
  verify = FALSE,
  trace = FALSE
)

6.2 Density covariate models

The next two models allow density to vary with spatial covariates on the mask. std_tri is standardised Terrain Ruggedness Index, and std_d2hydro is standardised distance to water. Standardised covariates are used because the model coefficients are easier to compare and optimisation is often more stable.

# Density varies with ruggedness (TRI)
m1 <- secr.fit(
  ch,
  detectfn = "HHN",
  mask = mask_model,
  model = list(D ~ std_tri, lambda0 ~ 1, sigma ~ 1),
  verify = FALSE,
  start = m0,
  trace = FALSE
)

# Density varies with distance to water
m2 <- secr.fit(
  ch,
  detectfn = "HHN",
  mask = mask_model,
  model = list(D ~ std_d2hydro, lambda0 ~ 1, sigma ~ 1),
  verify = FALSE,
  start = m0,
  trace = FALSE
)

The start = m0 argument gives the optimiser starting values from the null model. This is a practical way to make fitting related models more stable.

6.3 Detector covariate model

The next model keeps density constant but allows encounter rate to vary depending on whether the camera was placed in a valley or stream location.

# Encounter rate varies depending on whether detection is in a valley or stream
m3 <- secr.fit(
  ch,
  detectfn = "HHN",
  mask = mask_model,
  model = list(D ~ 1, lambda0 ~ valley_stream, sigma ~ 1),
  verify = FALSE,
  start = m0,
  trace = FALSE
)

Detector covariates describe camera locations, not animal activity-centre locations. They are therefore used with lambda0, not with D.

6.4 Combined models

The final two models combine density covariates and detector covariates. These models allow spatial variation in density and variation in encounter rate between types of camera sites.

# Combined model with a density covariate and a detector covariate
m4 <- secr.fit(
  ch,
  detectfn = "HHN",
  mask = mask_model,
  model = list(D ~ std_tri, lambda0 ~ valley_stream, sigma ~ 1),
  verify = FALSE,
  start = m1,
  trace = FALSE
)

# Combined model with a density covariate and a detector covariate
m5 <- secr.fit(
  ch,
  detectfn = "HHN",
  mask = mask_model,
  model = list(D ~ std_tri + std_d2hydro, lambda0 ~ cliff + valley_stream, sigma ~ 1),
  verify = FALSE,
  start = m4,
  trace = FALSE
)

7 Compare fitted models

8 Save fitted models

The fitted model objects are saved for the results tutorial, where they are used to produce AIC tables, abundance estimates, density maps, and covariate-effect plots.

save(
  m0, m1, m2, m3, m4, m5,
  file = output_file
)
